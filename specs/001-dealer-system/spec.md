# 经销商系统规格说明

## 目标

基于《OmniAI 产品需求文档（PRD V1.1）》先实现同一工程中的经销商系统。企业客户系统暂不实现，但代码结构需要预留后续扩展空间。

## 范围

本期实现经销商系统的可运行 MVP：

- 看板：首页指标、趋势图、排行榜、待办事项、系统公告。
- 模型：大模型、模拟试用、模型产品。
- 客户：我的客户、客户 API Key、消费记录、使用日志、客户报表。
- 财务：合同、账单。
- 管理：团队成员、角色管理、团队报表、个人中心。

## 数据要求

- 所有业务数据保存在浏览器本地 `localStorage`。
- 首次打开系统时写入一批相对真实的模拟数据。
- 支持对主要业务实体进行新增、查询、编辑、删除。
- 刷新页面后数据不丢失。
- 消费金额、成本、利润、客户活跃度等看板数据从本地数据计算，不写死为孤立展示数字。

## 界面要求

- 保持现有 AiDEA 风格：白色侧边栏、浅灰工作区、蓝色主色、圆角卡片。
- 经销商系统与企业客户系统在概念上分离；当前默认进入经销商系统。
- 菜单按 PRD 分组：看板、模型、客户、财务、管理。
- 表格页面需要具备搜索、新增、编辑、删除、重置等基础操作。

## 验收标准

- `pnpm test` 通过。
- `pnpm build` 通过。
- 页面可在本地启动后直接使用。
- 主要实体刷新后保留本地修改。
- 模拟数据包含合理企业名、模型名、金额、Token、合同、账单、团队成员和角色。
